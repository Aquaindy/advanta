Recommendations, Approvals & Autopilot

The safety system that governs every module. Agents find opportunities, turn them into recommendations, and nothing touches a live platform until it clears role gates and guardrails — or you've explicitly handed a category to Autopilot.

1Running agents — /agents

Agents are your AI growth team. Each runs against your real workspace data, saves its tasks + outputs, and emits recommendations.

  1. Open Agents and pick a card (Paid Ads, Creative Strategy, SEO audit, Landing-page audit, and more).
  2. Run now — the run is metered against your plan's agent-run quota.
  3. View last run — see every skill task (succeeded / failed / skipped), the raw outputs, and the recommendations it produced.

Agents have deterministic fallbacks, so they produce real output even without an LLM key.

2The Recommendations center — /recommendations

Everything agents find lands here, grouped by risk. Each card shows what was found, why it matters, the expected impact, the suggested action, and a link back to the agent run that produced it.

  1. Open one for the full detail, related platform/campaign, and history.
  2. Edit before applying (Admin+) — tighten the title, summary, impact, or action.
  3. Approve & apply — optionally toggle "apply to {platform} on approve" to execute the change immediately.
  4. Reject with a reason — it's recorded and the card closes out.

3Risk levels & role gating

Every recommendation carries a risk level, and the risk decides who's allowed to approve it:

RiskExamplesWho can approve
lowPause a past-end-date campaign; trim budget on a CPA spikeMarketer and up
mediumScale a winner's budget; publish a built draft ad set/adAdmin and up
highActions on stale/large campaignsOwner only

Editing a recommendation requires Admin+; reverting an execution requires Admin+.

4Execute, retry & revert

When a recommendation has an executable action, approving it (with auto-execute on) sends the change to the provider and records an execution:

  • What's captured — the provider, action type, exactly what was sent, the result, and the prior state (so it can be undone).
  • Retry — if an execution fails, retry it; an idempotency key prevents network retries from double-sending.
  • Revert (Admin+) — restore the prior state captured before the change. One-click undo for anything that went live.

5The three modes — /autopilot

Set how much autonomy the workspace grants. Off by default.

ModeBehavior
AdvisorAgents only analyze and recommend. No external actions, ever.
ApprovalAI proposes; a human approves before anything executes. Recommended default.
AutopilotRecommendations that pass all guardrails are auto-approved and logged as SYSTEM. Anything that fails a rule falls back to manual approval.

6Autopilot guardrails — Owner only

Autopilot will only ever act inside the limits you set:

  • Max daily spend increase & max daily spend total (hard caps)
  • Max % increase per change
  • Minimum recent conversions before any spend-up
  • Risk ceiling — low (safest) / medium / high
  • Allowed action types — an explicit whitelist (e.g. campaign.pause, campaign.update_budget)

Autonomous action tiers (each opt-in)

7Stop-loss & preview

🛑 Stop-loss — your emergency brake

One switch. While engaged, Autopilot refuses every recommendation regardless of guardrails. Flip it the moment something looks off.

The Autopilot page also has a Preview: for every open recommendation it shows whether it would auto-approve right now and why — so you can tune guardrails with full visibility before trusting them. A separate "run autonomous scan" lets the agents propose executable actions from live campaign signals on demand.

8Audit trail

Every sensitive action is logged — agent runs, approvals, rejections, edits, executions, reverts — with the actor (user or SYSTEM), IP address, and user agent. Open any recommendation to see its full history. Nothing happens invisibly.

?FAQ & tips

Will AI ever spend my money without me?

Only if you switch to Autopilot, whitelist the action type, and it passes every guardrail and the risk ceiling. Default is Approval — a human signs off on everything.

I approved something by mistake — can I undo it?

If it executed and captured a prior state, an Admin can Revert it to restore the previous setting on the provider.

Why is a recommendation "manual" even though Autopilot is on?

It failed a guardrail (cap, % limit, conversions threshold, risk ceiling, or it's not on the allowed-action whitelist). The Preview tells you which.

Does this apply beyond ads?

Yes — recommendations come from every module (SEO, Website, Traffic, Email). Approvals and audit logging are universal; executable provider actions are mainly ad-platform changes.
