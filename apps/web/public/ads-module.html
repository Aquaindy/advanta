⚡ 60-second quickstart

Settings → Integrations → Connect Google/Meta/LinkedIn → open Campaigns → Sync now → run the Paid Ads agent → review Recommendations → Approve. Done.

1Connect your ad platforms — external sources

This is the only place "connections" live. Go to Settings → Integrations (Admin role required). OAuth tokens are encrypted at rest; access tokens are never exposed to the browser.

  • Write access toggle — each ad platform card has an "Enable write access" checkbox. Leave it off for analytics-only (Advisor); turn it on to let AdGenieHQ run & manage ads. You can reconnect to change this later.
  • Disconnected = empty state, not fake data. Connect, then Sync to pull records.
  • Connecting/disconnecting is an Admin action and is written to the audit log.

2Campaigns dashboard — /campaigns

A unified, cross-platform view of every synced campaign.

  1. Sync now — pulls the latest campaigns from all connected ad platforms into AdGenieHQ.
  2. Filter by platform (Google / Meta / LinkedIn) and status (Active / Paused / Ended / Archived).
  3. Read the summary cards — total, active, paused, plus warnings: active campaigns with no budget and stale active (past end-date).
  4. Open a campaign to see detail, metrics, fees, and actions.

Heads up

Campaigns are primarily synced from the ad platforms — AdGenieHQ mirrors what's in your accounts. Use the builder (below) to draft and publish new structure as paused, then go live in approval.

3Campaign detail & actions — /campaigns/:id

On a campaign you can see budgets, schedule, identifiers, platform fees, metrics, and the raw provider payload. Sensitive actions are routed through the approval flow:

ActionWhat happensRisk / gate
PauseRequests a pause on the live platformLow · routed via approval
ResumeRequests resume on the live platformLow · routed via approval
Edit budgetInline form ($) to change the daily budgetMedium · routed via approval

Each action shows an approval status banner (pending → approved → executed, or rejected). Failed executions can be retried; succeeded ones can be reverted by an Admin (restores the prior state captured before the change).

4Ad-set & ad builder

Inside a campaign, the Ad structure section lets you build a launch-ready hierarchy and publish it safely.

  1. Add ad set — name, daily budget, optimization goal, locations, age range, interests. Saves as a draft.
  2. Add ad under an ad set — name, landing page URL, and an attached creative.
  3. Create creative — type, headline, primary text, description, CTA, image/video URL.
  4. Publish — pushes the ad set / ad to the live platform paused, as a recommendation that needs approval before it spends.

Hierarchy: Campaign → Ad set → Ad → Creative. Drafts you build never go live until approved.

5Creatives library — /creatives

  • Every creative the Creative Strategy agent generates, plus anything synced from your ad platforms.
  • Filter by type (search, responsive display, single image, video, carousel, UGC, other) and source (AI-generated, platform-synced, user-uploaded).
  • Click a creative to edit inline (headline, primary text, CTA, media URLs) before attaching it to an ad.
  • Platform-synced edits are local — push them back via the provider's own UI.

7Recommendations & approvals — /recommendations

Everything agents find lands here, grouped by risk. Each card shows what was found, why it matters, expected impact, the suggested action, and its source run.

  1. Review — open a recommendation for the full summary, impact, and audit trail.
  2. Edit before applying (Admin+) — tighten the title, summary, impact, or action.
  3. Approve & apply — optionally toggle "apply to {platform} on approve" to execute the change.
  4. Reject with a reason, or revert a succeeded execution to its prior state.

Role gating by risk: low → Marketer+   medium → Admin+   high → Owner. Every approve/reject/edit is audit-logged with actor, IP, and user agent.

8Autopilot & guardrails — /autopilot

Off by default. Choose the execution mode for the workspace:

ModeBehavior
Off / AdvisorAgents only analyze and recommend. No external actions.
ApprovalAI proposes; a human must approve before anything executes. (Recommended default.)
AutopilotRecommendations that pass all guardrails are auto-approved and audit-logged as SYSTEM. Anything that fails a rule falls back to manual approval.

Guardrails you set (Owner only)

  • Max daily spend increase & max daily spend total (caps)
  • Max % increase per change
  • Minimum recent conversions before scaling spend up
  • Risk ceiling — low (safest) / medium / high
  • Allowed action types — whitelist, e.g. campaign.pause, campaign.update_budget

Autonomous action tiers (each opt-in)

  • campaign.pause — pause past-end-date campaigns (spend-down, reversible)
  • campaign.update_budget — trim on CPA spikes / scale winners within caps
  • ad_set.create & ad.create — publish human-built drafts (paused)

🛑 Stop-loss

One switch on the Autopilot page. While active, Autopilot refuses every recommendation regardless of guardrails — your emergency brake. The Preview section always shows exactly what would auto-approve right now before you commit.

9Reports — /reports

  • Generate daily / weekly / monthly reports from connected data and saved AI outputs.
  • Download as PDF or CSV; email summaries supported.
  • Reports require connected data or completed agent runs — no data, no fabricated numbers.

?FAQ & tips

Why didn't a new integration appear on the Integrations page?

The Integrations page is only for OAuth platforms (Google, Meta, LinkedIn, GA4, Search Console, Stripe). Other channels live in the Traffic Module as a planning + tracking layer, because they have no API to connect to.

Can AI spend my money without me?

Only if you explicitly enable Autopilot, whitelist the action type, and it passes every guardrail. By default everything waits for human approval, gated by role and risk.

Do agents need an OpenAI key?

No. Every agent has a deterministic fallback, so you get real output even with no LLM configured. With a key, outputs become richer.

What's the safest way to start?

Connect read-only (write toggle off) → sync → run Paid Ads + Tracking & Attribution → read recommendations in Advisor mode. Turn on write access and approvals only when you're ready to act.
