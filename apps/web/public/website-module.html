⚡ 60-second quickstart

Website → add a landing page (or import from onboarding) → Run audit → read the three scores & fixes → spin up an A/B test to prove a change → install the snippet → Declare winner.

1Landing-page audits — /website

Track the pages that matter and audit each one with the landing_page_audit agent.

  1. Add a page — paste a URL (with an optional label), or Import from onboarding to pull the URLs you already gave us.
  2. Run audit — the agent fetches the live page, scores six conversion skills, and runs a real PageSpeed Insights query.
  3. Read the three scores — Conversion, Mobile UX, and Page Speed (each 0–100), with a per-skill breakdown and severity pills.
  4. Open the agent run for the full findings and prioritized recommendations.

The Conversion score is a weighted blend (CTA weighted highest, then above-the-fold, then trust / friction / clarity). Page speed under 50 raises a high-risk recommendation; 50–75 raises a medium one.

3Full-site conversion audit — via /agents

The website_audit agent checks your primary domain (from onboarding) for above-the-fold and basic on-page issues that hurt SEO, GEO, and conversion (title, meta description, headings, viewport, robots/noindex). Run it from the Agents dashboard. It audits a single URL — for per-page conversion scoring, use the landing-page audit above.

Ad tests launch paused

Ad A/B tests create their campaigns paused on the provider — you flip them to active explicitly, so nothing spends until you say so.

5Launch & install the snippet — landing-page tests

  1. Launch the test (draft → launched).
  2. Copy the snippet the test detail page generates — it's pre-filled with your test ID. Two modes:
    • Redirect mode — each variant is its own URL; visitors are routed to one.
    • In-place mode — one URL; the snippet shows/hides the variant blocks you've marked up.
  3. Add the conversion line on your success page (the one-liner shown on the same panel) so wins are counted.

Always copy from the app

Use the exact snippet shown on the A/B test detail page — it carries the correct script URL, test ID, and conversion call for your account. Assignment is sticky per visitor, and tracking errors are swallowed so they can never break your page.

6Read results & pick a winner

  • Per variant — visits, conversions, and conversion rate, fed by the snippet's anonymous assign/convert events.
  • Statistics — relative lift, confidence interval, p-value, a suggested winner, and an "underpowered" flag when the sample is too small to trust.
  • Record metrics manually for ad tests, or sync from GA4 (if Google Analytics is connected) to pull sessions + conversions.
  • Declare winner when you're confident, then Archive the test.

7Reports — /reports

Generate a Daily, Weekly, or Monthly roll-up of everything your AI growth team did. Each report pulls real workspace data and downloads as PDF or CSV.

Inside: agent activity, open/approved/rejected & high-risk recommendations, campaign stats, SEO keywords, landing-page scores, A/B test counts, content drafts, outreach, and Growth DNA readiness — only the sections you actually have data for.

?FAQ & tips

Does the audit crawl my whole site?

No — a landing-page audit scores the single URL you give it (that's what makes the per-page scores precise). For your primary domain's basic on-page health, run the full-site website_audit from Agents.

Is the page-speed score real?

Yes — it's a live Google PageSpeed Insights query (mobile strategy), not an estimate. It works without an API key; a key just raises the rate limit.

Why isn't my A/B test counting conversions?

Make sure you (1) launched the test, (2) installed the snippet on the variant page(s), and (3) added the conversion line on your success page. Copy all three from the test detail page.

Will ad A/B tests spend money automatically?

No. Ad variants are created paused on the provider; you activate them explicitly. See the Ads module for approval & Autopilot guardrails.
